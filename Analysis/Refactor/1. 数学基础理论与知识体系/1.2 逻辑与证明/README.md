## 1.2.1 命题逻辑与推理

命题逻辑是研究命题及其联结、推理规则和真值关系的形式系统，是数学和计算机科学推理的基础。

- **定义**：
  - 命题是可以判断真假的语句。
  - 命题逻辑通过联结词（如"与" "或" "非" "蕴含" "等值"）构造复合命题。
  - 形式语言：命题变元、联结词、括号组成的表达式。

- **基本结构**：
  - 命题变元（$p, q, r, \ldots$）
  - 基本联结词：
    - 合取（与）：$\land$
    - 析取（或）：$\lor$
    - 非（否定）：$\lnot$
    - 蕴含：$\to$
    - 等值：$\leftrightarrow$
  - 真值表：列举所有命题组合的真假值

- **推理规则**：
  - 演绎推理（如假言推理、三段论、归谬法）
  - 等值变换（德摩根律、双重否定、分配律等）
  - 归结法、合取范式/析取范式
  - 证明方法：直接证明、反证法、穷举法

- **典型应用**：
  - 电路设计、自动推理、AI 规则系统、形式化验证
  - 计算机程序的条件判断、控制流

- **行业趋势**：
  - 可视化推理工具、自动化定理证明器
  - 形式化方法在软件/硬件验证中的应用
  - AI 逻辑推理与可解释性

- **多表征与可视化建议**：
  - 真值表、推理树、逻辑电路图、公式化证明
  - Mermaid 代码示例：

```text

  A["p"] --> B["¬p"]
  A --> C["q"]
  B --> D["p ∨ q"]
  C --> D
```

## 1.2.2 谓词逻辑与证明

谓词逻辑（First-Order Logic, FOL）扩展了命题逻辑，引入了对象、属性和量词，能够表达更复杂的数学和现实世界结构。

- **定义**：
  - 谓词逻辑以对象、谓词（属性/关系）、量词（全称 $\forall$、存在 $\exists$）为基本元素。
  - 形式语言：变元、谓词符号、函数符号、量词、联结词组成的表达式。

- **基本结构**：
  - 个体变元（$x, y, z, \ldots$）
  - 谓词符号（$P(x), Q(x, y)$ 等）
  - 量词：全称量词 $\forall x$，存在量词 $\exists x$
  - 公式的自由变元与约束变元
  - 结构化表达：如 $\forall x\, (P(x) \to Q(x))$

- **推理与证明方法**：
  - 量词规则（全称引入/消去、存在引入/消去）
  - 归纳证明、反证法、结构归纳
  - 统一与归结、Herbrand 基、Skolem 形式
  - 证明树、自然演绎、序列演算

- **典型应用**：
  - 知识表示、数据库查询（SQL、Datalog）
  - 形式化验证、自动定理证明、模型检测
  - AI 领域的知识推理、语义理解

- **行业趋势**：
  - 高阶逻辑、模态逻辑、描述逻辑的发展
  - 自动化证明工具、知识图谱推理
  - AI 领域的可解释推理与符号-神经融合

- **多表征与可视化建议**：
  - 量词树、推理流程图、结构化证明表
  - Mermaid 代码示例：

```text
flowchart TD
  X["x"] --> P["P(x)"]
  X --> Q["Q(x)"]
  P --> R["P(x) ∧ Q(x)"]
  Q --> R
```

## 1.2.3 数学归纳法与递归证明

数学归纳法和递归证明是处理递归结构、无穷过程和离散对象的核心方法，广泛应用于数学、计算机科学等领域。

- **定义**：
  - 数学归纳法是一种证明自然数集上命题成立的基本方法，分为基础步骤和归纳步骤。
  - 递归证明是对递归定义对象（如树、列表、图等）性质的证明方法。

- **基本结构**：
  - 基础步骤（Base Case）：证明命题在最小元素（如 $n=0$ 或 $n=1$）成立。
  - 归纳假设（Inductive Hypothesis）：假设命题在 $n=k$ 时成立。
  - 归纳步骤（Inductive Step）：证明 $n=k \Rightarrow n=k+1$ 时命题成立。
  - 结构归纳法：对树、图等递归结构的归纳证明。

- **证明方法**：
  - 普通归纳法、强归纳法、结构归纳法
  - 递归定义与递归证明的配合
  - 反证法与归纳法结合

- **典型应用**：
  - 算法正确性证明、组合恒等式、递归公式
  - 数据结构（树、链表、图）性质证明
  - 计算复杂度分析、自动机理论

- **行业趋势**：
  - 自动化归纳证明、结构化归纳工具
  - AI 归纳推理、程序合成与验证
  - 归纳法与模型检测、定理证明器集成

- **多表征与可视化建议**：
  - 归纳树、递归流程图、结构化证明表、伪代码
  - Mermaid 代码示例：

```text
flowchart TD
  Base["基础步骤"] --> IH["归纳假设"]
  IH --> Step["归纳步骤"]
  Step --> QED["结论成立"]
```

## 1.2.4 形式系统与元逻辑

形式系统与元逻辑关注逻辑与数学理论的公理化、推理规则、系统自洽性与表达能力，是现代数学基础与自动化证明的核心。

- **定义**：
  - 形式系统由符号、语法规则、公理、推理规则组成，用于形式化描述和推导数学对象。
  - 元逻辑研究形式系统本身的性质，如一致性、完备性、可判定性等。

- **基本结构**：
  - 符号与语法：字母表、公式、推导规则
  - 公理系统：如皮亚诺公理、ZF 集合论公理、希尔伯特系统
  - 推理规则：如 Modus Ponens、归结、自然演绎
  - 元定理：哥德尔不完备定理、紧致性定理、可判定性定理

- **典型应用**：
  - 定理证明器（如 Coq、Lean、Isabelle）、形式化验证
  - 计算机科学中的类型系统、编译器、自动机理论
  - 数学基础、元数学、逻辑哲学

- **行业趋势**：
  - 可验证计算、自动化元推理、AI 辅助证明
  - 形式化方法在软件/硬件安全中的应用
  - 交互式定理证明、可解释 AI 逻辑

- **多表征与可视化建议**：
  - 公理系统结构图、推理规则表、元定理依赖图
  - Mermaid 代码示例：

```text

  A["公理"] --> B["推理规则"]
  B --> C["定理"]
  C --> D["元定理"]
```
